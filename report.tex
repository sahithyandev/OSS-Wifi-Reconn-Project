\section{Introduction}

Wi-Fi reconnaissance is the process of collecting information about nearby wireless networks before conducting security assessments or penetration testing activities. It helps identify wireless access points, security configurations, connected devices, and possible vulnerabilities within a wireless environment.

In this assignment, the Aircrack-ng suite was used to perform Wi-Fi reconnaissance. Aircrack-ng is a collection of tools used for wireless network monitoring, packet capturing, and security analysis.

\subsection{Tools Used}

\begin{itemize}
    \item \textbf{airmon-ng} — enables monitor mode on the wireless adapter.
    \item \textbf{airodump-ng} — scans nearby wireless networks and captures packets.
    \item \textbf{aireplay-ng} — performs deauthentication attacks for capturing WPA/WPA2 handshake packets.
\end{itemize}

The reconnaissance process focused on gathering information such as SSIDs, BSSIDs, channels, encryption types, and signal strengths of nearby wireless networks.

\subsection{Background}

\subsubsection{802.11 Wireless Communication}

Wi-Fi devices communicate using the IEEE 802.11 family of standards over shared radio frequencies. All frames broadcast over the air are visible to any nearby device operating in \emph{monitor mode}, which allows passive capture without association to any network. 802.11 defines three frame types: management frames (beacons, probe requests, authentication, deauthentication), control frames, and data frames. Management frames in WPA and WPA2 networks are sent in plaintext and carry no authentication, which is the root cause of the deauthentication vulnerability exploited later in this assignment.

\subsubsection{WPA, WPA2, and WPA3}

\textbf{WPA} (Wi-Fi Protected Access, 2003) was introduced as an emergency replacement for the broken WEP standard. It uses the TKIP cipher, which wraps RC4 with per-packet key mixing. TKIP is now considered cryptographically weak and is deprecated.

\textbf{WPA2} (IEEE 802.11i, 2004) replaced TKIP with the AES-based CCMP cipher, which is significantly stronger. Two authentication modes exist: Personal (PSK — a shared passphrase) and Enterprise (802.1X with a RADIUS server). WPA2-Personal is the most common deployment and is the target of offline passphrase-cracking attacks once the four-way handshake is captured.

\textbf{WPA3} (2018) introduced Simultaneous Authentication of Equals (SAE), which replaces the PSK handshake with a Diffie--Hellman-based exchange that provides forward secrecy and is resistant to offline dictionary attacks. WPA3 also mandates Protected Management Frames (PMF/802.11w), preventing the deauthentication attack described in Section~\ref{sec:deauth}.

\subsubsection{The WPA/WPA2 Four-Way Handshake}

When a client associates with a WPA2 network, a four-message exchange derives the session key (PTK) from the PSK and two random nonces supplied by the access point and the client. No plaintext password is transmitted, but the handshake messages include enough material for an attacker to attempt offline dictionary or brute-force attacks: a candidate passphrase is used to recompute the PTK and the MIC (message integrity code), and if the MIC matches the captured value, the passphrase is correct. Capturing this handshake is therefore the primary goal of the targeted capture and deauthentication steps below.

\textbf{Observations:}

\begin{itemize}
    \item \textbf{32 networks} used WPA2 encryption; \textbf{1 network} (Redmi A5) used WPA3.
    \item \textbf{4 networks} were fully open (no encryption), including the ``Redmi A1+'' hotspot.

    \item Several networks advertised both WPA2 and the older WPA standard simultaneously, indicating mixed-mode or legacy configurations.
    \item Multiple mobile hotspots were visible, including devices named ``Thavanathan's iPhone'', ``Galaxy M0298c8'', and ``TimBaTTuK's S25 Ultra''.

\textbf{Interpretation:}

The scan results illustrate how much a passive observer can learn without authenticating to any network. Four open networks expose all transmitted traffic in plaintext, leaving users vulnerable to credential theft and session hijacking. The 32 WPA2 networks provide reasonable protection with strong passphrases, but several ISP and IoT devices broadcast mixed WPA/WPA2 configurations, which reintroduce the weaker TKIP cipher and can be exploited. Only one network used WPA3, underscoring its slow real-world adoption. The seven hidden SSIDs offer no practical security benefit: access points still broadcast beacon frames and were captured trivially by \texttt{airodump-ng}. IoT devices (cameras, dash cameras, bridge units) advertise distinctive SSIDs and typically run outdated firmware, making them attractive entry points for network intrusion.

\subsection{Deauthentication Attack}
\label{sec:deauth}

A deauthentication attack was performed against an isolated private Wi-Fi access point (a personal mobile hotspot) to force connected clients to reconnect and generate WPA handshake packets. The target was selected because it was under personal control and no third-party devices were affected.

\subsubsection{Broadcast Deauthentication}

An initial broadcast deauthentication attack was sent to all clients associated with the target access point:

\begin{lstlisting}
sudo aireplay-ng -0 5 -a A6:8E:DC:82:EB:7B wlp2s0mon
\end{lstlisting}

This sent five deauthentication frames to the broadcast address, requesting all connected clients to disconnect from the access point.

\subsubsection{Directed Deauthentication}

A directed attack targeting a specific client was then attempted. An initial attempt failed because the monitor interface was locked to channel 13 while the access point operated on channel 6:

\begin{lstlisting}
sudo aireplay-ng -0 5 -a A6:8E:DC:82:EB:7B -c 14:5A:FC:81:7E:1D wlp2s0mon
\end{lstlisting}

Running \texttt{airmon-ng check kill} resolved the channel lock issue. A subsequent directed deauthentication against client \texttt{24:EB:16:56:E1:CC} succeeded:

\begin{lstlisting}
sudo aireplay-ng -0 5 -a A6:8E:DC:82:EB:7B -c 24:EB:16:56:E1:CC wlp2s0mon
\end{lstlisting}

\texttt{aireplay-ng} reported 63 out of 67 ACKs for the sent frames, confirming the client received and acted upon the deauthentication packets.

A continuous attack (\texttt{-0 0}) was then run for approximately five minutes to observe sustained deauthentication behaviour. ACK counts varied across packets, indicating intermittent reconnection attempts by the client.

\subsubsection{Fake Authentication Attempt}

A fake authentication request was also sent to the access point to test association behaviour:

\begin{lstlisting}
sudo aireplay-ng -1 0 -a A6:8E:DC:82:EB:7B -c 24:EB:16:56:E1:CC wlp2s0mon
\end{lstlisting}

The open-system authentication succeeded, but the subsequent association request was denied with status code 12 (\emph{wrong ESSID or WPA}), confirming that WPA2 protection was enforced and association without valid credentials was not possible.

\textbf{Interpretation:}

The attack demonstrated a fundamental weakness in the 802.11 protocol: management frames such as deauthentication are unauthenticated by design, so any station can forge and send them without possessing network credentials. The high ACK rates (63 out of 67) confirm that the frames were received and acted upon by the client. WPA3 networks with Management Frame Protection (MFP) are not vulnerable to this attack because their management frames are cryptographically signed; the absence of WPA3 across nearly all detected networks means this technique would work against the vast majority of access points observed.

\subsection{WPA Handshake Capture}

During reconnection triggered by the deauthentication attack, WPA handshake packets were captured successfully. The handshake notification appeared in the top-right corner of the \texttt{airodump-ng} window:

\begin{lstlisting}
WPA Handshake: A6:8E:DC:82:EB:7B
\end{lstlisting}

\textbf{Interpretation:}

Capturing the four-way WPA handshake is the prerequisite for offline passphrase cracking. Once the handshake is stored in a capture file, an attacker can attempt dictionary or brute-force attacks against it using tools such as \texttt{aircrack-ng} or \texttt{hashcat} without any further interaction with the target network. The risk to the network owner is therefore not limited to the moment of capture — a weak passphrase can be recovered long after the event. The collected information could potentially be used for unauthorised network access, password cracking, traffic monitoring, device tracking, and denial-of-service attacks.

\subsection{Passphrase Cracking Attempt}

To illustrate the offline cracking phase, \texttt{aircrack-ng} was run against the captured handshake file using the \texttt{rockyou.txt} wordlist, one of the most widely used password dictionaries in security assessments:

\begin{lstlisting}
aircrack-ng -w /usr/share/wordlists/rockyou.txt capture-01.cap
\end{lstlisting}

The attack ran for approximately 20 minutes running \textbf{14,344,391} candidate passphrases at a sustained rate of approximately \textbf{11,780 keys per second}. The correct passphrase was \textbf{not found} within the rockyou list.

The failed dictionary attack demonstrates that a strong, non-guessable passphrase is an effective defence against offline dictionary attacks, even when an attacker has successfully captured the four-way handshake. An attacker would need to escalate to larger wordlists, rule-based mutations, or full brute-force — each of which increases the time and computational cost by orders of magnitude.

\section{Summary and Conclusion}

This assignment demonstrated the process of conducting Wi-Fi reconnaissance using the Aircrack-ng suite. The tools \texttt{airmon-ng}, \texttt{airodump-ng}, and \texttt{aireplay-ng} were used to scan nearby wireless networks, gather information, and capture WPA/WPA2 handshake packets.

Across six scanning sessions conducted on 2026-05-16 (12:27–12:54), 39 unique access points were identified. Their SSIDs, BSSIDs, channels, encryption configurations, and signal strengths were collected passively without authenticating to any network. The activity also demonstrated how deauthentication attacks can force clients to reconnect and generate handshake packets for offline analysis.

The findings highlighted several real-world wireless security concerns: the presence of open hotspots, wide deployment of legacy mixed WPA/WPA2 configurations, near-absent WPA3 adoption, inadequately obscured IoT devices, and the fundamental insecurity of unauthenticated 802.11 management frames.

\subsection{Recommendations}

The following measures can improve wireless security:

\begin{itemize}
    \item Use WPA2 or WPA3 encryption
    \item Avoid open wireless networks
    \item Use strong, unique passwords
    \item Disable WPS functionality
    \item Regularly update router firmware
    \item Monitor connected devices regularly
\end{itemize}

In conclusion, Wi-Fi reconnaissance is an important part of wireless security assessment because it helps identify vulnerabilities and security weaknesses before attackers can exploit them.
